Frequently asked questions

Info on the TNT library and how to get started.

Who are the developers of TNTgo?

TNTgo is powered by the TNT library, developed in the Quantum Systems Engineering (QSE) group at the University of Oxford. The TNTgo interface was jointly developed in the (QSE) group and with Thomas Grujic. The main developers of the TNT library are Sarah Al-Assam, Stephen Clark and Dieter Jaksch. We have also received support from the Software Engineering Group at the Rutherford Appleton Lab and from NAG.

Why is there a limit on the system size?

The calculations are all run on the QSE group cluster in the University of Oxford. System sizes are limited so control the load on these systems. If you wish to simulate larger systems, you can download the TNT library from CCPForge, and run calculations locally.

Why should I use conserved quantities in my simulations?

For ground state calculations, doing so ensures that the state found will be a state having a definite total spin or boson number, which will not be the case e.g. if using the chemical potential to tune the number of bosons.

For time evolution calculations where you know your Hamiltonian conserves total spin/boson number, doing so allows the algorithm to take advantage of the global symmetries present and so can substatially speed up run times.

What do I do if my calculation ends in an error?

Please first check your simulation parameters to make sure they are physically sensible. If you cannot see an error please email support@tensornetworktheory.org with the calculation id and we will look into this.
